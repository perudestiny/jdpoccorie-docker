# 🐧 Instalación de Docker en Linux

En Linux existen **varias formas de instalar Docker**, y la elección dependerá principalmente de:

- La distribución Linux que utilices
- La versión del sistema operativo
- La arquitectura del procesador
- Si deseas instalar **Docker Desktop** o únicamente **Docker Engine**

Distribuciones comunes:

- Ubuntu
- Debian
- Fedora
- Arch Linux
- Otras variantes

Cada una posee ligeras diferencias en su proceso de instalación.

---

## 🧠 Docker en Linux: Concepto Importante

Actualmente, **Docker Desktop para Linux** utiliza una herramienta de virtualización interna que permite tener una experiencia similar a:

- Windows
- macOS
- Linux

Esto significa que:

✅ Los comandos
✅ El comportamiento
✅ El flujo de trabajo

son prácticamente iguales sin importar el sistema operativo.

---

## ⚖️ Docker Desktop vs Docker Engine

Antes de continuar, debes conocer las dos opciones disponibles:

| Opción | Incluye | Uso recomendado |
|---|---|---|
| Docker Desktop | Interfaz gráfica + Docker Engine | Principiantes |
| Docker Engine | Solo terminal | Usuarios Linux avanzados |

👉 Puedes seguir el curso usando cualquiera de las dos opciones.

# 🐧 Instalación Manual de Docker Engine en Linux

En esta sección finalizaremos la instalación de Docker en Linux **de forma manual**.

> 💡 Aunque se le llama instalación manual, en realidad utilizaremos
> herramientas automatizadas proporcionadas por Docker.

Esta instalación es útil cuando:

- Docker Desktop no es compatible con tu sistema
- Prefieres trabajar únicamente desde terminal
- Usas servidores Linux sin entorno gráfico
- Quieres una instalación más ligera

---

## ⚠️ Importante Antes de Comenzar

Los pasos mostrados están basados en **Ubuntu**.

Si utilizas otra distribución Linux:

👉 Consulta siempre la documentación oficial de Docker
👉 Sigue los pasos específicos para tu sistema

Cada distribución puede variar ligeramente.

---

# 🧹 Paso 1 — Eliminar Instalaciones Previas

Primero eliminamos versiones antiguas de Docker (si existen).

Abre una terminal nueva y ejecuta:

    sudo apt-get remove docker docker-engine docker.io containerd runc

Si no había instalaciones previas, no ocurrirá nada. ✅

---

# 📦 Paso 2 — Configurar el Repositorio Oficial

Docker se instalará desde su repositorio oficial.

Ejecuta los siguientes comandos uno por uno.

---

## Actualizar paquetes del sistema

    sudo apt-get update

---

## Instalar dependencias necesarias

    sudo apt-get install ca-certificates curl gnupg

---

## Agregar clave GPG oficial de Docker

    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

---

## Agregar repositorio Docker

    echo \
    "deb [arch=$(dpkg --print-architecture) \
    signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

---

# 🔄 Paso 3 — Actualizar Repositorios

Ahora actualizamos nuevamente:

    sudo apt-get update

Si aparece algún error con `apt-get`, simplemente vuelve a ejecutar el comando.

---

# 🐳 Paso 4 — Instalar Docker Engine

Instalamos todos los componentes necesarios:

    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Esto instalará:

- Docker Engine
- CLI de Docker
- Container Runtime
- Docker Compose
- Herramientas adicionales necesarias

---

# 🧪 Paso 5 — Probar la Instalación

Ejecuta:

    sudo docker run hello-world

Si todo está correcto verás:

    Hello from Docker!

✅ Esto confirma que Docker funciona correctamente.

---

# ⚠️ Problema Común: Uso de sudo

Notarás que debes escribir `sudo` cada vez que uses Docker.

Ejemplo:

    sudo docker ps

Esto es incómodo y **no es la práctica recomendada**.

Vamos a solucionarlo.

---

# 🔧 Paso 6 — Ejecutar Docker sin sudo

## Crear grupo docker

    sudo groupadd docker

(Si ya existe, aparecerá un mensaje indicando eso).

---

## Agregar tu usuario al grupo docker

    sudo usermod -aG docker $USER

---

## Aplicar cambios de grupo

    newgrp docker

También puedes cerrar sesión y volver a entrar.

---

# ✅ Paso 7 — Verificación Final

Ahora ejecuta sin sudo:

    docker run hello-world

Si funciona correctamente:

🎉 Ya puedes usar Docker normalmente.
